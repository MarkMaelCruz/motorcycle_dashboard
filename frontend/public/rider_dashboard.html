System Pipeline

The interface follows the actual system workflow: sensing, recording, live streaming, upload, classification, and analytics.

Architecture
1
Sensing

Motorcycle dynamic inputs and rider body position metrics are captured through the sensor network.

2
RPi4 Acquisition

The Raspberry Pi timestamps, synchronizes, validates, and formats the incoming data.

3
Parallel Operation

Complete dataset is recorded locally while selected dynamic inputs are streamed live.

4
Upload

After session end, the complete local session log is uploaded to the backend server.

5
ML Classification

Backend preprocessing, feature extraction, and multi-class riding control classification are executed.

6
Analytics

Performance metrics, trends, strengths, weaknesses, and feedback are displayed.

Defines the operating parameters for data acquisition and dashboard display.
